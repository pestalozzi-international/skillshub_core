System Overview
What SkillsHub is and how the pieces fit together

SkillsHub is a Pestalozzi International programme that takes applicants from initial selection through vocational training, soft skills, remedial education and workplace attachment — then tracks them as alumni. The system has two interfaces:

  • This portal (/skillshub/admin/…) — day-to-day admin work: student management, attendance, and data overview.
  • The Desk (/app ↗) — Frappe's back-office, used for creating programmes, classes, enrolments and detailed record editing.

Key record types

RecordWhat it isWhere you create it
SH Student The core person record. Holds name, DOB, contact details, guardian info, and photo. The student's ID (e.g. SH.26.0001) comes from this record. Desk
SH Applicant Created when someone submits the online application form. Reviewed before a Student record is made. Online form / Desk
SH Class A specific run of a course — e.g. VT-R1A. Holds dates, instructors, and max strength. Desk
SH Enrolment Links one student to one class. Tracks attendance totals, status, and whether forms have been submitted. Desk
SH Academic Year Container for a cohort year, used for filtering in reports. Desk

Programme structure

Programmes run in a defined order. A student typically passes through:

  1. Mindset Camp — residential orientation
  2. Edulution (Remedial) — literacy and numeracy
  3. Soft Skills Programme — life and work skills
  4. Vocational Training — trade or practical skill
  5. Attachment — workplace placement

Students on Path A or Path B may follow different sequences. The path is stored on the student's enrolment and synced back to the student record automatically.

Logging In
Accessing the admin portal and the desk

Admin portal

Navigate to /skillshub/admin/students. You will be redirected to the Frappe login page if you are not already signed in. Log in with your Pestalozzi staff email and password.

ℹ️ You need the PI Admin, SH Admin, or System Manager role to see the admin portal. Contact your system administrator if you get a "Not permitted" error.

The Desk

Click Desk ↗ in the top navigation to open the Frappe back-office in a new tab. The same login credentials apply.

Student portal

Students access their own portal at /skillshub/s/. They log in with their Student ID and Date of Birth — no password is needed.

The Student Journey
From application to alumni — end-to-end overview
  1. Application submitted — via the online portal (/skillshub/apply) or entered manually as an SH Applicant record in the Desk. Status is Submitted.
  2. Review & interview — the admissions team reviews the applicant in the Desk. After interview, the record is updated with notes and outcome.
  3. Student record created — once accepted, a new SH Student record is created in the Desk. The system assigns a Student ID automatically (see next section).
  4. Enrolment created — the student is enrolled in their first class via an SH Enrolment record linking them to an SH Class. Status is Enrolled.
  5. Baseline form — the student completes the Baseline Assessment via the student portal. This captures their starting point.
  6. Programme delivery — classes run. Attendance is recorded daily using the portal Attendance page.
  7. Feedback forms — at the end of each programme segment, the student completes the relevant feedback form in the portal.
  8. Graduation — when all enrolments are completed, the student's status is changed to Alumni in the Desk.
  9. Graduate Feedback — alumni can complete the Graduate Feedback form at any time to report on their post-programme outcomes.

Issuing Student IDs
How IDs are generated and how to share them

How the ID is generated

Every SH Student record is assigned an ID automatically when it is saved for the first time. The format follows the naming series:

FormatSH.YY.####
ExampleSH.26.0001
YYLast 2 digits of current year
####Sequential 4-digit counter

The counter resets each year, so the first student created in 2027 will be SH.27.0001. You do not need to enter the ID manually — the system assigns it on save.

✅ The ID is case-sensitive when students log in to the portal. It is always uppercase: SH.26.0001 — never sh.26.0001.

Creating a student record in the Desk

  1. Open the Desk (/app ↗) and search for SH Student in the search bar, or navigate to ZM SkillsHub → SH Student.
  2. Click New (top right).
  3. Fill in at minimum: First Name, Last Name, Date of Birth, and Gender. These fields are required for portal login.
  4. Enter the student's mobile number and personal email if known.
  5. Set Status to Student.
  6. Link the Applicant Record if one exists.
  7. Click Save. The ID (e.g. SH.26.0047) is assigned immediately and shown in the title bar and the Full Name and ID field.
⚠️ Do not change the Naming Series field. Doing so will break the sequential numbering and could create duplicate or out-of-order IDs.

Sharing the Student ID with the student

Students need their ID to log into the portal. Communicate it clearly — by letter, text message, or verbally during orientation:

  • Tell them their Student ID (e.g. SH.26.0047) and that it is case-sensitive.
  • They will also need their Date of Birth (which they know) — this acts as their password.
  • The portal login page is at /skillshub/s/.
ℹ️ You can view all student IDs from the Students page or print physical student cards from the Student Cards page.

Enrolments
Linking students to classes

An SH Enrolment connects one student to one class. A student will have one enrolment per programme segment they participate in.

Creating an enrolment

  1. In the Desk, navigate to SH Enrolment → New.
  2. Select the Student — type the student ID or name to search.
  3. Select the Class (e.g. VT-R1A). The course, programme, and path fields populate automatically.
  4. Set the Enrolment Date.
  5. Leave Status as Enrolled.
  6. Save.

Enrolment statuses

StatusMeaning
EnrolledStudent is actively attending this class.
CompletedStudent finished this programme segment successfully.
DroppedStudent left before completion. Set an exit reason.
✅ The student's Programme Path field is synced automatically from their most recent active enrolment — you don't need to set it manually.

Attendance
Recording daily attendance via the portal

Attendance is recorded per class, per day. Use the portal Attendance page — no Desk access needed.

  1. Go to Admin → Attendance.
  2. Select the Class from the dropdown.
  3. Select the Date.
  4. The roster loads showing all enrolled students. Mark each student Present or Absent.
  5. Click Save Attendance. The totals on each enrolment record update automatically.
ℹ️ Attendance totals (Sessions Present / Absent / Rate) are stored on each SH Enrolment record and visible in the Programme Overview and on each student's profile.

Programme Overview
Monitoring data health and completion rates

Portal tree view

The Programme Overview page shows a collapsible tree: Intake Year → Cohort → Path → Programme → Class. Each class leaf shows enrolment counts, baseline count, and feedback counts for that class's programme type.

Use the filter bar to narrow by year, cohort, path, programme, course, or course run. Click Expand All to see all students within each class.

Desk report

For exports, sorting, and per-student detail, use the full desk report:

📊 Open Full Report ↗
  • Class Summary view — one row per class. Colour-coded percentage columns show baseline and feedback completion rates.
  • Student Detail view — one row per enrolment with individual baseline ✓/✗ and feedback ✓/✗ flags. Filter by student name to find a specific person.

Use the Export button (top right of the desk report) to download as CSV or Excel.

Graduating to Alumni
Changing a student's status at programme completion

When a student has completed all required programme segments, change their status to Alumni. This unlocks the Graduate Feedback form in their portal.

How to graduate a student

  1. Find the student in the Students page and open their profile.
  2. Click the Edit Record tab.
  3. Change Status to Alumni.
  4. Click Save Changes.

Alternatively, update the status directly in the Desk via SH Student.

ℹ️ Alumni students no longer see in-programme feedback forms. Only the Graduate Feedback form appears in their portal. They can continue to log in as normal with their Student ID and Date of Birth.

Before graduating

Completing all enrolments does not automatically set the student to Alumni — you must do this manually. Before graduating, make sure all their SH Enrolment records are set to Completed.
